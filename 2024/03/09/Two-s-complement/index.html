有符号数(signed magnitude)

直觉上，拿二进制数的最高一位作为符号位区分正负就好了。最高位为1代表负数，0代表负数。如第四列

针对Signed magnitude我们有一些观察，尽管它简单易用，但存在一些问题

  • 出现了两个0，+0和-0
  • 最致命的问题是，在进行算数运算时，有符号数会出错，(+3） + （-3）=111+011=(1)010=-2,不论去不去掉最高的进位都不等于0 。同一个数和他的相反数相加居然不等于0，这显得十分荒谬。

2’s complement

寻找二补数

我们设想一下，如果有一个二进制的正数x,只要找到一个？。满足x + ? = 0，这个问号不就是我们要找的满足计算功能的负数吗?
下面开始寻找满足条件的”?”。我们采用倒推的方式。以4bit的二进制数为例，我们知道

  • 1111 + 1 = 1 0000 ->忽视进位后的最高位(挖坑)= 0000。
  • x + （x的1’s complement） = 1111。例如，1111+0000=1111.1010+0101=1111

综上: x + （x的1’s complement) + 1 = 0,最终x的-x等于x的1’s complement + 1 ，也就是x的二补数。对于一个有符号的二进制数，取反加一就得到相应的相反数，对一个正数取反加一就得到对应的相反数了。

关于0

已知0000为+0,取反加一后，为10000，只要我们忽视多出的进位，零的二补数仍然为本身0000，满足了0的唯一性(填坑)。

二补数的好处

二补数弥补了有符号数错误计算的不足，例如-3 + 3 ≠ 0。改进相反数相加不等于零后再表示负数，在实现加法的同时也获得了减法。例如：16-8 = 16 + (-8)

question : 不考虑溢出的情况，两数相加一定正确，拓展到减法后，计算结果为什么也正确呢？

二补数的计算本质

我们实际要证明的是，X-Y或X+(-Y)可以用X加上Y的2的补码完成(8bit为例)。我们的可利用结论是Z = X - Y= X + (11111111-Y) + 1 。

这就证明了，在正常的加法规则下，可以利用2的补码得到正数与负数相加的正确结果。换言之，计算机只要部署加法电路和补码电路，就可以完成所有整数的加法
